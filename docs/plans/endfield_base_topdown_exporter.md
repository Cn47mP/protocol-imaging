# 协议映射

> 集成工业基地全景快照成像

## 方案文档

> **目标**：基于 PC 端屏幕截图生成一张可导出的基地俯视总览图，方便查看设备摆放和暗管走向。

---

## 1. 背景

游戏当前没有直接的基地俯视导出功能，人工拼接截图成本高、效率低、容易错位。需要做一个外部工具，仅通过屏幕捕获和图像处理，把基地多张视角图拼成一张总览图。

---

## 2. 目标

### 2.1 核心目标
- 采集 PC 游戏窗口画面
- 支持连续截图
- 自动或半自动拼接为大图
- 导出清晰的 PNG 总览图
- 支持基础标注，方便查看设备位置
- 叠加暗管走向图层，标注地下管线走向

### 2.2 范围边界
- 聚焦外部屏幕捕获与图像拼接
- 不处理游戏客户端内部数据
- 不做自动操作游戏角色
- 不追求一开始就全自动无误拼接

---

## 3. 方案总览

### 3.1 总体思路
工具分为四层：

1. **截图层**：从游戏窗口外部采集画面
2. **对齐层**：对连续截图做位置校正
3. **拼接层**：把多张图合成大图
4. **导出层**：输出 PNG，并可选叠加标注

### 3.2 推荐实现路径
先做 **半自动 MVP**：
- 用户手动控制游戏视角移动
- 工具负责连续截屏
- 工具提供手动锚点校正
- 自动生成初版拼图
- 用户少量修正后导出

这样最稳，风险最低，也最容易先跑通。

---

## 4. 技术选型

### 4.1 语言与框架
- **Python**：原型开发快，图像处理生态成熟
- **OpenCV**：图像拼接、特征匹配、透视变换
- **PySide6** 或 **PyQt**：桌面界面
- **mss** / **dxcam**：Windows 屏幕捕获

### 4.2 可选增强
- **PaddleOCR**：识别界面文字、设备名称
- **numpy**：图像数组处理
- **Pillow**：简单导出与标注

## 5. 功能拆分

### 5.1 截图模块
职责：
- 捕获指定窗口或屏幕区域
- 连续采样帧
- 保存原始截图供后处理

建议功能：
- 选择窗口
- 设置截图频率
- 暂停 / 继续
- 导出原始帧序列

### 5.2 预处理模块
职责：
- 裁剪固定 UI 区域
- 去除黑边
- 统一缩放
- 提升后续匹配成功率

建议功能：
- 固定比例裁切
- 亮度归一化
- 去噪

### 5.3 对齐模块
职责：
- 找到相邻截图之间的重叠区域
- 计算位移或透视关系
- 将图像放到统一坐标系

实现策略：
- 优先尝试 ORB / AKAZE 特征匹配
- 匹配失败时允许用户手工指定锚点
- 对低质量重叠区域保留人工修正入口

### 5.4 拼接模块
职责：
- 根据对齐结果生成总图
- 合并重叠区域
- 处理透明边界与融合

建议方式：
- 先做平面拼接
- 再考虑更复杂的多图融合

### 5.5 标注模块
职责：
- 在总图上加图层标记
- 标出设施名称、编号、说明文字
- 支持开关图层

### 5.6 导出模块
职责：
- 导出 PNG
- 可选导出分层文件
- 保存项目配置，便于下次继续编辑

### 5.7 暗管标注

背景：
- 终末地基地的水管、电缆等暗管（地下管线）在游戏内没有导出/显示走向的功能
- 需要在俯视图上叠加暗管路径线，便于用户手动标注和查看管线走向

职责：
- 在总览图上添加可编辑的管线层
- 支持手绘管线路线（鼠标拖拽画线）
- 支持标注文字说明（管道类型/走向/连接设备）
- 支持独立开关管线层的显示
- 支持导出含管线和不含管线的版本

建议功能：
- 自由绘制：多点折线/贝塞尔曲线
- 多种颜色区分管线类型（上水/下水/电缆/气路等）
- 标注锚点吸附到已有设备位置
- 撤回/重绘
- 管线层与底图分别导出

## 6. MVP 版本定义

### 6.1 MVP 必须具备
- 选择游戏窗口
- 连续截图
- 手动触发停止
- 4 点手动校正
- 拼成一张总图
- 导出 PNG

### 6.2 MVP 暂不做
- OCR 自动识别
- 自动识别设备类别
- 自动检测基地边界
- 复杂图层编辑
- 云同步

---

## 7. 数据流

```text
游戏窗口画面
  -> 截图帧
  -> 预处理
  -> 对齐
  -> 拼接
  -> 标注
  -> 导出 PNG
```

如果后续要做项目保存：

```text
截图帧 + 锚点 + 对齐参数 + 标注信息
  -> 项目文件（JSON）
  -> 可重新打开继续编辑
```

## 10. 验收标准

### 10.1 功能验收
- 能选择游戏窗口并稳定截图
- 能生成一张完整总览图
- 导出的 PNG 可正常放大查看
- 总图中设备位置基本正确

### 10.2 质量验收
- 不依赖注入或内存读取
- 拼接结果无明显大面积错位
- 操作步骤不超过普通用户可接受范围

---

## 11. 风险与对策

### 风险 1：视角变化导致拼接失败
对策：
- 降低自动化期待
- 支持手动锚点校正
- 限制截图时的镜头运动方式

### 风险 2：UI 干扰拼接
对策：
- 预处理阶段裁掉固定 UI
- 只保留地图有效区域

### 风险 3：重复纹理导致特征点误配
对策：
- 结合人工锚点
- 允许局部修正
- 不追求纯自动零误差

### 风险 4：导出图过大
对策：
- 支持分辨率限制
- 支持分块拼接
- 最终再统一导出

## 13. 设定归属

协议映射属 **终末地工业体系**。

原因：
- 管线（水管/暗管/电缆）是**基地基础设施**的一部分，是集成工业系统的日常运维对象
- 终末地工业的核心职能是建设、运行、维护和记录基地的全部工业设施
- 宏山科学院的领域在源石技艺解析、符文结构、理论实验方向，不涉及工业图纸归档
- 管线走线标注属于工业档案管理，与"协议核心→协议传送→协议回收"的终末地技术链一脉相承

```text
终末地工业 — 协议映射
```
