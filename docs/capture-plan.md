# 采集方案：终末地基建俯视快照规划

> 更新时间：2026-05-29
> 项目：协议映射

## 概述

协议映射的目标是对终末地游戏内的基建基地进行俯视截图采集，并将多帧拼接为完整全景图。

当前 P0/P1 路线已经合入：工具可以自动检测游戏窗口、用 Win32 输入模拟移动视角、按蛇形网格采集截图，并在拼接阶段执行模糊检测和羽化融合。本文档保留采集网格的估算和真实游戏校准项。

## 关键问题与当前结论

### 1. 基地区域面积有多大？

终末地的 AIC（自动化工业综合体）基地采用 **基于格子的平铺系统**：

- 核心 AIC 区可通过 AIC 管理员逐步扩建，每个阶段增加可建造格子数
- 四号谷地（Valley IV）/武陵等区域有各自的扩建路线
- 已知建筑参考尺寸：
  - 灌装机/打包机 ≈ 6 格宽
  - PAC 输入口：9 格边可放 7 个端口
  - 仓库卸货器：3 个输入口占 9 格（0.33 端口/格）
- 满扩建后，核心 AIC 区范围预计 **30×40 至 40×50 格**（目测级，待实测校准）

> **待校准项**：进入 AIC 模式后从最高俯视角度截图并数一下可见格子数。

### 2. 截图横竖各多少张？

以 **1920×1080 分辨率、相机拉到最远俯视** 为基准：

| 参数 | 估算值 |
|------|--------|
| 单帧可见范围 | ~8×5 格（横×纵，估算，取决于相机高度） |
| 满扩建基地范围 | ~36×40 格（含外界边缘） |
| 横向需拍张数 | 36÷8×0.8(重叠) ≈ **6 张** |
| 纵向需拍张数 | 40÷5×0.8(重叠) ≈ **10 张** |
| **总张数** | **≈ 60 张**（含额外边界覆盖） |

> **待校准项**：进入游戏后，从 AIC 模式最高俯视角度拍一张，数一下水平/垂直各能看到多少格，然后反推最终网格。

### 3. 自动移动位置怎么做？

#### 方案 A：固定网格扫描（已实现，当前推荐）
```
视角锁定为垂直俯视（相机旋转角度固定）
以基地一角为起点，按 Z 字路径逐行扫描：

 行1: ▸▸▸▸▸▸
 行2: ◂◂◂◂◂◂
 行3: ▸▸▸▸▸▸
 行4: ◂◂◂◂◂◂

每步移动距离 = 单帧视野的 80%（留 20% 重叠用于全景拼接）
```

当前实现位置：

- `app/control/game_controller.py`：负责鼠标拖拽、滚轮缩放、前台检测
- `app/capture/auto_capturer.py`：负责蛇形扫描、截图、模糊检测、进度回调
- `app/ui/main_window.py`：负责自动检测窗口、选择网格预设、启动/取消自动采集

#### 方案 B：锚点/特征点定位（已具备基础能力，待真实样本调参）
- 先拍一张大步幅全景（20% 重叠）
- 用特征匹配（ORB/SIFT）自动检测实际重叠量
- 基于匹配结果修正移动步长

#### 方案 C：手动辅助（保留为兜底）
- 用户手动移动相机到每个网格位置
- 按快捷键采集单帧
- 工具负责导入和拼接

### 4. 校准步骤（必须先做）

1. 进入游戏 AIC 模式
2. 将相机拉到最高、转至垂直俯视
3. 截图一张
4. 数视野中完整可见的格子数（水平 × 垂直）
5. 记录并填入下面的占位值，更新此文档
6. 计算：基地总格数 ÷ 单帧可见格数 = 所需拍数

## 后续实现优先级

1. **真实场景校准**：进入 AIC 模式实测单帧覆盖格数，修正默认网格和移动距离
2. **校准工具**：在 GUI 中添加"手动校准"模式，能标注单帧覆盖格子数
3. **网格规划**：根据校准结果自动生成拍摄路径网格
4. **采集鲁棒性**：处理窗口遮挡、游戏未前台、拖拽失败、拼接失败等异常
5. **标注增强**：完善暗管/设备标签编辑体验
